# エラーとは。

エラー = 異常。

今回、エラーとは何か、なぜエラーを出すのかと聞かれた時に

なぜか答えられなかったので言語化。

色々 claude とかと壁打ちしてしっくりきたのでそれを記事にしてみた。

# そもそもエラーを出す目的

そもそもエラーを出す目的はなんであるか。

Rails では色々なメソッドが処理を続ける上で、

何らかの問題があって処理を続けることがでいなくなった場合例外を発出する。

大抵それを rescue で補足して log を出すなり、上位に再伝播するなどの対応をするのが普通。

ただここで 1 つ疑問がある。

```
エラーを出す目的とは何だろうか。
```

「エラー = まずい」ということはエンジニアではなくても

何となく「悪い、いけないこと」であるのは理解している。

ただ、なぜ出すのか、と聞かれたら、その時答えられない自分がいた。

今回、色々なものを参考に、自分なりに再定義してみた。
その内容を掘り下げて、ソフトウェア開発におけるエラーハンドリングの話にまで持っていけたら良い。

# 再定義

> エラーとは、リクエストされた処理を正常に完遂できない状態を表現する言語要素
> エラーを出す目的とはその処理を正常に完遂できない時にエラーを宣言することで、Rails の異常系フローに処理を以上するため。
